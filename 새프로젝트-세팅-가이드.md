# 새 프로젝트 세팅 가이드 — Cowork 세션에 붙여넣기용

아래 내용을 새 프로젝트의 Cowork 첫 세션에 그대로 붙여넣으세요.

---

## 붙여넣기 시작 ↓

```
나는 코딩 초보자야. 새 프로젝트를 시작하는데, 프로젝트 관리 체계를 먼저 세팅하고 싶어.
아래 구조대로 파일들을 만들어줘. 내가 프로젝트 이름과 내용을 알려줄게.

## 1. 만들어야 할 파일 4개

### (1) CLAUDE.md — Claude Code 지침서
프로젝트 루트에 생성. 내용:
- 프로젝트 개요 1줄
- 절대 규칙 (예: 빌드 깨진 코드 push 금지, console.log 금지, 한 번에 하나의 작업만 등)
- 폴더 구조 (app/, docs/, content/, archive/ 등)
- 문서 업데이트 규칙: 코드 작업 완료 시 반드시 업데이트할 파일 4개 명시
  - CLAUDE.md 헤더 날짜
  - docs/CHANGELOG.md
  - session-context.md
  - docs/NEXT_SESSION_START.md
- 세션 종료 체크리스트
- 참조 파일 경로 테이블
- 내 스타일: 코딩 초보자, 기술 설명 간결하게, 작업은 직접 해줘야 함, 한국어 소통, 코드/기술 용어는 영어

### (2) session-context.md — 프로젝트 맥락 파일
프로젝트 루트에 생성. 내용:
- 프로젝트 개요 (서비스 정의, 핵심 전략)
- 현재 TODO (우선순위별 P0~P5)
- 완료된 세션 히스토리 (매 세션마다 추가)
- 핵심 수치 (기능 수, 페이지 수, 사용자 수 등)
- 세션 업데이트 지침: "이 파일에 없는 숫자를 임의로 만들지 말 것"

### (3) docs/CHANGELOG.md — 변경 이력
docs/ 폴더에 생성. 내용:
- 세션별로 추가/변경/삭제 기록
- 각 항목에 커밋 해시 포함
- 가장 최근 세션이 맨 위

### (4) docs/NEXT_SESSION_START.md — 다음 세션 시작 가이드
docs/ 폴더에 생성. 내용:
- 현재 상태 요약 (핵심 수치)
- 가장 최근 세션 요약 (커밋, 변경사항)
- 다음 할 일 (우선순위별)
- 미해결 사항
- 참조 파일 경로

## 2. 하네스(Harness) 시스템 — 자동 강제 규칙

### (1) 세션 종료 시 자동 검증 Hook
`.claude/hooks/stop-reminder.sh` 파일 생성:
- 세션 종료할 때 자동으로 4개 문서의 헤더 날짜가 오늘인지 검증
- git에 커밋 안 된 변경사항이나 push 안 된 커밋이 있으면 경고
- 이건 "부탁"이 아니라 "강제" — 날짜가 안 맞으면 ❌ 표시

스크립트 내용:
```bash
#!/bin/bash
TODAY=$(TZ=Asia/Seoul date '+%Y-%m-%d')
echo ""
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
echo "🔒 세션 종료 하네스 검증"
echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"

# CHANGELOG.md 날짜 검증
if [ -f "docs/CHANGELOG.md" ]; then
  DATE=$(head -2 docs/CHANGELOG.md | grep -oP '\d{4}-\d{2}-\d{2}' | head -1)
  [ "$DATE" != "$TODAY" ] && echo "❌ CHANGELOG.md: $DATE (오늘: $TODAY)" || echo "✅ CHANGELOG.md OK"
fi

# session-context.md 날짜 검증
if [ -f "session-context.md" ]; then
  DATE=$(head -2 session-context.md | grep -oP '\d{4}-\d{2}-\d{2}' | head -1)
  [ "$DATE" != "$TODAY" ] && echo "❌ session-context.md: $DATE (오늘: $TODAY)" || echo "✅ session-context.md OK"
fi

# NEXT_SESSION_START.md 날짜 검증
if [ -f "docs/NEXT_SESSION_START.md" ]; then
  DATE=$(head -2 docs/NEXT_SESSION_START.md | grep -oP '\d{4}-\d{2}-\d{2}' | head -1)
  [ "$DATE" != "$TODAY" ] && echo "❌ NEXT_SESSION_START.md: $DATE (오늘: $TODAY)" || echo "✅ NEXT_SESSION_START.md OK"
fi

# CLAUDE.md 날짜 검증
if [ -f "CLAUDE.md" ]; then
  DATE=$(head -2 CLAUDE.md | grep -oP '\d{4}-\d{2}-\d{2}' | head -1)
  [ "$DATE" != "$TODAY" ] && echo "❌ CLAUDE.md: $DATE (오늘: $TODAY)" || echo "✅ CLAUDE.md OK"
fi

# git 상태 검증
if git status --porcelain 2>/dev/null | grep -q '^'; then
  echo "❌ 커밋 안 된 변경사항 있음!"
else
  AHEAD=$(git rev-list --count @{u}..HEAD 2>/dev/null || echo "0")
  [ "$AHEAD" -gt 0 ] && echo "❌ push 안 된 커밋 ${AHEAD}개!" || echo "✅ git OK"
fi

echo "━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━"
exit 0
```

### (2) CLAUDE.md에 하네스 규칙 섹션 추가
```markdown
## 🔒 하네스 규칙 (자동 강제 — 부탁이 아닌 시스템)

### 세션 종료 시 자동 검증
- Hook이 4개 문서 헤더 날짜를 자동 검증
- 오늘 날짜가 아니면 ❌ → 반드시 업데이트 후 push

### 가비지 컬렉션 (세션 시작 시 필수)
- 매 세션 시작 시 session-context.md의 TODO 섹션 점검
- 완료된 항목이 TODO에 남아있으면 즉시 제거
- 1주일 이상 지난 "대기 중" 항목은 날짜 갱신 필요 여부 확인

### 문서 4개 날짜 일치 규칙
- 코드 작업이 있는 세션에서는 4개 문서 헤더 날짜가 반드시 오늘이어야 함
- CLAUDE.md, docs/CHANGELOG.md, session-context.md, docs/NEXT_SESSION_START.md
```

## 3. 세션 루틴 규칙

### 세션 시작 시:
1. CLAUDE.md 읽기 (규칙 확인)
2. session-context.md 읽기 (맥락 + TODO)
3. docs/NEXT_SESSION_START.md 읽기 (최신 상태)
4. 가비지 컬렉션: TODO에서 완료된 항목 정리

### 세션 종료 시:
1. 코드 변경 있으면 → 4개 문서 헤더 날짜 오늘로 업데이트
2. CHANGELOG.md에 이번 세션 변경사항 추가
3. session-context.md에 이번 세션 완료 블록 추가
4. NEXT_SESSION_START.md 최신 상태로 업데이트
5. git push
6. Notion에 세션 로그 기록 (사용하는 경우)

### 핵심 원칙:
- "로그 없으면 미완료" — 빌드 성공해도, 테스트 통과해도, 기록 없으면 미완료
- "session-context.md에 없는 숫자 만들기 금지" — 근거 없는 수치 사용 금지
- "한 번에 하나의 작업만" — 멀티태스킹 금지

---

이 구조로 내 프로젝트를 세팅해줘. 프로젝트 정보는 다음과 같아:

**프로젝트 이름**: [여기에 프로젝트명]
**한줄 설명**: [여기에 프로젝트 설명]
**기술 스택**: [여기에 사용 기술]
**현재 상태**: [새로 시작 / 진행 중 — 어디까지 했는지]
**폴더 위치**: [프로젝트 폴더 경로]
```

## 사용법

1. 새 프로젝트 Cowork 세션 열기
2. 위 내용에서 맨 아래 `[여기에 ...]` 부분만 내 프로젝트에 맞게 채우기
3. 통째로 붙여넣기
4. Claude가 4개 파일 + hook 스크립트를 자동으로 생성해줌
5. 이후 세션부터는 `docs/NEXT_SESSION_START.md 읽고 가비지 컬렉션 실행해줘` 한 줄이면 OK
