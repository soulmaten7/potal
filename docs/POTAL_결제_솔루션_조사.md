# POTAL 글로벌 결제 솔루션 조사 보고서

**작성일**: 2026년 3월 8일
**대상**: LemonSqueezy 거절에 따른 대체 결제 수단 분석
**적용 대상**: POTAL (B2B SaaS, 한국 기반, 글로벌 API/위젯 판매)

---

## 1. 현황 분석

### LemonSqueezy 거절 이유
- "we cannot approve your store application for Lemon Squeezy" - 명확한 이유 미제시
- 추정 원인: B2B API 과금 모델에 대한 정책 불일치, 또는 한국 사업자에 대한 제약

### POTAL의 특수성
- **구조**: B2B SaaS (이커머스 셀러 → 계약 → API/위젯 사용)
- **과금 방식**: 사용량 기반 API 과금 (Transaction-based, User-based, 또는 Tiered)
- **고객**: 전 세계 이커머스 셀러 (240개국)
- **통화**: 다중 통화 지원 필요
- **특수성**: 관세/세금 규제 정보 포함 (금융 규제 관련)

**Stripe를 위한 선택지**:
1. **Stripe Atlas** (미국 법인 설립)

   - 더 복잡하고 비용이 높음

**결론**: Stripe의 가장 현실적인 옵션은 Stripe Atlas로 미국에 법인 설립 후 가입하는 것. 하지만 세무 복잡성과 초기 비용이 있음.

### 2.2 Paddle (MoR 모델)

| 항목 | 내용 |
|------|------|
| **한국 사업자 지원** | ✅ 가능 (국내외 모두 계약 가능) |
| **글로벌 결제** | ✅ 220개국+ 지원 |
| **수수료** | 5% + $0.50 (고정) |
| **MoR 모델** | ✅ Yes (VAT/GST 자동 처리) |
| **세금 처리** | ✅ 완전 자동 (MoR) |
| **SaaS/API 과금** | ✅ Subscription + Usage 지원 |
| **세금계산서** | ❌ 불가능 (B2B 한국 고객 제약) |
| **결제 수단** | 카드, 계좌이체(EUR/GBP/USD), PayPal, Apple Pay, Google Pay, Alipay |

**수수료 상세**:
- 기본: 5% + $0.50
- 국제/PayPal: 별도 추가 없음 (단일 정액요금)
- 환전 시 추가 1% 손실
- **한국 고객 판매 시**: 약 5-7% + $0.50 (최종 7.5% 이상)

**한국 사업자 이용 경험**:
- 한국 스타트업 다수 사용 중 (예: 인블로그)
- 월 1,000만 원 이상 매출에서 비용이 부담 시작

**주요 문제**:
- B2B 한국 고객이 세금계산서 요청할 경우 대응 불가능
- VAT 자동화되지만, 한국 부가세(VAT) 처리는 별도 필요

**결론**: 한국 사업자가 글로벌 고객 대상으로 사용하기에 **가장 빠르고 간단한 옵션**. 초기 런칭에는 최적. 높은 수수료가 문제.

## 3. POTAL에 최적의 솔루션: 종합 분석

### 3.1 추천 순위

#### 🥇 1순위: Stripe + Stripe Atlas (미국 법인)

**이유**:
- 가장 낮은 수수료 (2.9% + $0.30)
- SaaS API 과금에 최적화 (가장 유연한 API)
- 글로벌 표준 (240개국 고객에게 신뢰도 높음)
- 장기 성장 시에도 확장성 우수

**실행 계획**:
```
1. Stripe Atlas 가입 (초기 $250-500, 연간 $400-600)
2. 미국 Delaware LLC 설립
3. Stripe US 계정 개설
4. POTAL (한국) → Stripe US → 글로벌 고객
   (또는 POTAL이 Stripe US 자회사 역할)
5. 세무: 한-미 조세조약 활용, 한국 세무사 상담
```

**비용**:
- 초기: Stripe Atlas $250-500
- 연간: 미국 LLC 유지비 $400-600, 한국 세무신고 추가 비용
- 거래 수수료: 2.9% + $0.30

**타임라인**: 1-2주 내 Stripe US 계정 개설 가능

---

#### 🥈 2순위: Paddle (즉시 런칭용)

**이유**:
- **즉시 시작 가능** (한국 사업자 직접 가입)
- MoR 모델로 글로벌 세금/컴플라이언스 자동화
- API 과금 지원 (SaaS 구독 + 사용량 기반)
- 한국 스타트업 다수 성공 사례

**한계**:
- 높은 수수료 (5-7.5%, 최종 환전 포함)
- B2B 한국 고객의 세금계산서 대응 불가
- 월 1,000만 원 이상 매출 시 수수료 부담 증가

**적합한 시나리오**:
- 초기 MVP/런칭 (매출 확인 단계)
- 해외 고객 중심 (B2C 위주)
- 빠른 시장 진입 필요

**비용**: 5-7.5% + 환전 손실 1%

**타임라인**: 수일 내 런칭 가능 (가입 심사 1-2일)

---

#### 🥉 3순위: FastSpring (SaaS 전문, 조건부)

**이유**:
- SaaS/API 과금에 가장 특화
- 2,500개 SaaS 회사 신뢰
- MoR 모델로 글로벌 세금 완전 자동화

**전제조건**:
- ❓ 한국 사업자 지원 여부 **확인 필수**
  → support@fastspring.com 직접 문의 필요

**조건 충족 시 고려**:
- 매출이 충분히 크면 SaaS 최적화 가치 발휘
- 수수료는 Paddle보다 높을 것으로 예상 (8.5% 이상)

**타임라인**: 한국 지원 확인 후 1-2주 (SaaS 특화 심사 더 엄격할 수 있음)

### 3.2 단기 vs 장기 전략

#### 단기 (1-6개월, MVP/초기 런칭)
```
Paddle 즉시 시작
├─ 장점: 빠른 런칭, 복잡도 낮음
├─ 비용: 5-7.5% (높지만 확인된 매출만)
└─ 목표: 해외 고객 확보, 매출 모델 검증
```

#### 중기 (6-12개월, 성장 단계)
```
Stripe + Stripe Atlas 병행 준비
├─ 병행: Paddle + Stripe US (하이브리드)
├─ 목표: 해외 고객 → Stripe US로 마이그레이션
└─ 이점: 수수료 절감 확인 (Paddle 5-7% → Stripe 2.9%)
```

#### 장기 (12개월+, 규모화)
```
Stripe 단독 운영
├─ 한국 자회사: POTAL KR (기술/영업)
├─ 미국 자회사: POTAL US (결제/계약)
└─ 이점: 최저 수수료, 최고의 유연성, 글로벌 표준
```

## 4. 구체적 실행 로드맵

### 4.1 선택안 A: Stripe + Stripe Atlas (권장)

**STEP 1: Stripe Atlas 가입 (1-2일)**
```
1. stripe.com/atlas 접속
2. 개인정보 + 한국 거주 주소 입력
3. $250 결제 (프로모션 링크 필요) 또는 $500
4. Delaware LLC 자동 설립 (2-3일)
5. EIN 번호 발급 (1주)
```

**STEP 2: Stripe US 계정 개설 (1일)**
```
1. 미국 LLC 정보로 Stripe 계정 생성
2. 사업 정보 입력 (B2B SaaS - API)
3. 은행 정보 입력 (한국 계좌 또는 해외 송금 계좌)
4. 심사 (1-2일)
5. 승인
```

**STEP 3: 세무 설정 (지속)**
```
1. 한국 세무사 상담 (한-미 조세조약, 세무신고 구조)
2. 한국: POTAL KR에서 미국 Stripe US로 로열티/라이선스비 지급
   → 한국 부가세 과세 제외 (면세)
3. 미국: Stripe US 소득 → 한국 거주자 배당금 → 한-미 조세조약 적용
4. 연간 세무신고: 미국 (1040-NR) + 한국 (종합소득세)
```

**STEP 4: API 연동 (1주)**
```
1. Stripe Billing API 학습
2. POTAL API → Stripe Billing 연동
3. 구독 + 사용량 기반 과금 구현
4. 테스트 (자동 인보이싱, 결제, 환전)
```

**총 타임라인**: 3-4주 (세무 설정 제외)

## 5. 세금/규제 고려사항

### 5.1 한국 부가세 (VAT/GST)

**현황**:
- POTAL이 한국 사업자이므로 기본적으로 한국 부가세 과세 대상
- 해외 고객 판매 = 영역 외 공급 = 한국 부가세 **면세** (조건부)

**조건**:
```
해외 고객 판매 면세 기준:
1. 고객이 한국 거주자 아닐 것
2. 판매 기록 확실할 것 (IP, 주소, 결제 수단 등)
3. 정산 통화가 외화일 것 (KRW 아님)
4. 정부 보고: 면세 공급 신고
```

**Paddle의 VAT 자동화**:
- Paddle이 MoR로 글로벌 VAT/GST 징수 및 정산
- 한국은 VAT가 아니라 부가세이므로 Paddle의 자동화 범위 제외
- **결과**: 한국 기준 면세 신고 필요 (세무사 상담)

**Stripe의 경우**:
- VAT 자동화 없음 (고객사 책임)
- 하지만 Stripe의 자세한 거래 기록으로 세무 신고 용이

---

### 5.2 한국 사업자의 미국 법인 세무

**Stripe Atlas 선택 시 필수**:
```
한국 거주 개인 → 미국 LLC (POTAL US) 소유

세무 구조:
1. 미국: POTAL US 소득세 (연방 21% + 주세 0-5%)
2. 한국: 미국 LLC 배당금 수령 시 종합소득세

한-미 조세조약:
- 미국 LLC의 한국 거주자 소유 → 투명성 규칙 적용
- 미국에서 일단 과세 → 한국에서 외국세액공제
- 효과: 이중 과세 방지, 한국 세율로 최종 조정
```

**필요한 조치**:
1. 한국 세무사 상담 (조세조약 전문가)
2. 미국 CPA 고용 (연간 $2,000-5,000)
3. 한국 세무신고: Form 3010 (해외 사업 신고)
4. 미국 세무신고: Form 1040-NR (비거주자 소득세)

**비용**: 연간 $2,500-6,500 (세무 대행)

### 5.3 한국 고객의 세금계산서 문제

**Paddle/MoR 모델의 한계**:
- B2B 한국 고객이 세금계산서 요청 → 발급 불가
- 이유: Paddle이 MoR이므로 한국 세금계산서 발급 불가능

**해결 방안**:
1. **Paddle + 국내 PG (토스 등) 병행**
   - 한국 고객: 토스 (세금계산서 가능)
   - 해외 고객: Paddle (MoR)
   - 복잡하지만 가능

2. **Stripe + 한국 법인 구조**
   - 한국 법인 POTAL KR이 모든 고객 계약
   - 토스 또는 PG로 한국 고객 결제
   - Stripe로 해외 고객 결제
   - 가장 정확한 세무 구조

3. **Paddle만 사용 + 수동 세금계산서**
   - 한국 고객이 세금계산서 요청 → 수동 발급 (수수료 감안)
   - 간단하지만 관리 복잡
