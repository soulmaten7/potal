# POTAL Daily Content Posting Skill

이 스킬은 POTAL에 대한 일일 콘텐츠를 생성한다. 하나의 핵심 메시지를 잡고, 3개 플랫폼 각각의 톤과 포맷에 맞게 변환하는 것이 핵심이다.

## 플랫폼 구조

| 구분 | 플랫폼 | 방식 |
|------|--------|------|
| **매일 자동 생성** | LinkedIn, DEV.to, Medium | 이 스킬이 매일 생성 |
| **브랜드 채널** | X, Threads, Instagram, 디스콰이어트 | 은태님이 직접 작성 (특별한 순간에) |

브랜드 채널은 이 스킬의 자동 생성 대상이 아니다. 일요일 주간 소재 준비 시 "브랜드 채널 포스팅 아이디어 3개"를 제안하는 것으로 충분.

## 브랜드 가이드라인

이 가이드라인은 모든 콘텐츠의 뼈대다. 여기서 벗어나면 POTAL이 아니다.

### WHY
**국경이 장벽이 되면 안 된다. 그래서 무료다.**

POTAL은 크로스보더 커머스에서 관세·세금·규정이라는 보이지 않는 장벽을 없애는 인프라다. 경쟁사가 월 $1,500~$4,000을 받는 걸 POTAL은 $0에 140개 기능 전부 제공한다. 이건 할인이 아니라 철학이다.

### 브랜드 성격
**시끄럽게 존재하되, 제품으로 증명한다.**

매일 모든 플랫폼에 나타난다. 조용히 숨어있지 않는다. 하지만 과장하지 않는다. 숫자와 팩트로 말한다. "우리가 최고!"가 아니라, "경쟁사 $4,000/월, POTAL $0. 140개 기능 전부."

### 포지셔닝: FREE
FREE는 두 가지 의미를 동시에 담는다:
- **무료 (Free of charge)** — 140개 기능, $0, Forever
- **자유 (Freedom)** — 국경의 장벽에서 자유로워지는 것

이 이중 의미를 콘텐츠 안에서 자연스럽게 활용한다. 억지로 둘 다 넣을 필요는 없다. 맥락에 따라 하나만 써도 된다.

### 톤
- **팩트 기반** — 과장 없이. 실제 숫자, 실제 기능, 실제 비교
- **자신감** — 변명하지 않는다. 무료라서 품질이 낮다는 인상을 주지 않는다
- **매일 어디에나** — 빈도가 곧 존재감. 하루도 빠지지 않는다
- **한글 병기** — 영문 콘텐츠에는 반드시 한글 번역 포함

### 비주얼 방향
- 깔끔, 여백, 미니멀
- "무료 Apple" 느낌 — 프리미엄하면서 무료
- 데이터 비주얼(비교 차트, 숫자 강조)이 효과적

### 🚫 절대 금지 표현 (이 목록에 해당하면 무조건 삭제하고 다시 써라)

**과장 단어 — 영어:**
insane, revolutionary, mind-blowing, game-changer, groundbreaking, disruptive, crazy, incredible, unbelievable, amazing, stunning, jaw-dropping, epic, legendary, killer, beast, monster, ultimate

**과장 단어 — 한국어:**
역대급, 혁명적, 미친, 충격적, 끝판왕, 레전드, 대박, 압도적, 신세계

**구걸 표현 — 절대 사용 금지:**
- "Follow me", "Follow us", "팔로우 해주세요"
- "Share this", "RT please", "공유 부탁"
- "Drop a comment", "Comment below", "댓글 달아주세요"
- "Like if you agree", "좋아요 부탁"
- "Subscribe", "구독해주세요"
- "Tag a friend", "친구 태그"
- "Don't miss out", "놓치지 마세요"
- "Link in bio"

**이모지 규칙:**
- 문단 시작마다 이모지 붙이기 금지
- 한 포스트에 이모지 최대 3개까지만
- ✅❌🔥🚀💰 같은 "마케팅 이모지" 나열 금지
- 이모지 없이 쓰는 게 기본. 필요할 때만 최소한으로

**CTA는 자연스럽게만:**
- 허용: "potal.app", "Check it out at potal.app", "Try it: potal.app"
- 금지: 위의 구걸 표현과 결합된 CTA

## 핵심 콘텐츠 소스 (POTAL이 가진 것)

콘텐츠는 전부 POTAL에 대한 것이다. 외부 프로젝트, 개인 이야기, 관련 없는 업계 뉴스를 쓰지 않는다. POTAL의 기능, 데이터, 경쟁 비교, 사용 사례에서만 뽑는다.

### 숫자 팩트 (session-context.md에서 확인, 임의 수치 금지)
- 140개 기능, $0 Forever Free
- 240개국 지원
- 경쟁사: Avalara $1,500+/월 (31기능), Zonos $4,000+/월 (31기능)
- HS Code 분류: 595 GRI 규칙, AI 비용 $0
- 7개국 정부 API 연동
- AI 에이전트: ChatGPT, Claude MCP, Gemini 지원
- SDK: npm (potal-sdk), PyPI (potal)
- 플랫폼: Shopify, WooCommerce, BigCommerce, Magento

### 140개 기능 카테고리 (토픽 소스)
`app/features/features-guides.ts` 파일에서 상세 내용 확인 가능:
- **Core Engine (15)**: HS Code 분류, 관세 계산, Total Landed Cost, FTA, Confidence Score 등
- **Country Intelligence (7)**: 240개국 관세율, 제재 스크리닝, 수입규정 등
- **AI Agent (6)**: ChatGPT, Claude MCP, Gemini, Copilot 등
- **Platform (18)**: Shopify, WooCommerce, BigCommerce, Magento, API 위젯 등
- **Data & Analytics (12)**: 실시간 환율, 배송비, HS 이력 등
- **Security (5)**: RLS, Rate Limiting, CORS 등
- **Developer (15)**: REST API, SDK, Webhook, Sandbox 등
- **Legal & Compliance (6)**: De Minimis, 원산지 규정 등
- **Support (8)**: 다국어 지원, 커뮤니티 등
- **Business (5)**: White-label, 파트너 프로그램 등
- 그 외 Web, Integrations, Marketing, Shipping 등

## 토픽 선택 시스템

매일 다른 주제를 다뤄야 한다. 같은 얘기를 반복하면 지루하다.

### 토픽 카테고리 (로테이션)
1. **기능 딥다이브** — 140개 중 하나를 골라 구체적으로 설명
2. **경쟁 비교** — Avalara/Zonos/Global-e 등과 가격·기능 비교. 팩트만.
3. **사용 사례** — "한국 셀러가 미국에 판매할 때 POTAL이 뭘 해주는가"
4. **업계 뉴스 + POTAL** — De Minimis 폐지, EU 면세 폐지 등 규제 변화와 POTAL 연결
5. **빌딩 저니** — 코딩 0 → 140기능 1달, AI 바이브코딩 스토리
6. **개발자용** — API 사용법, SDK 예시, MCP 연동
7. **데이터/인사이트** — 240개국 관세 데이터에서 나오는 흥미로운 사실

### 토픽 선택 방법
1. `content/social-media/daily-posts/` 폴더에서 최근 7일간 생성된 파일 확인
2. 최근에 다루지 않은 카테고리 우선 선택
3. 해당 주의 트렌드 (예: De Minimis 관련 뉴스가 있으면 4번 카테고리)
4. 선택한 토픽을 파일명에 카테고리 번호로 기록 (예: `2026-04-03_cat2_avalara-comparison.md`)

## 플랫폼별 가이드 + 노출 최적화 (매일 자동 생성 3개)

하나의 핵심 메시지를 쓰고, 각 플랫폼에 맞게 변환한다. 전부 같은 글을 복붙하는 게 아니다.
각 플랫폼의 발견(discovery) 시스템을 이해하고 최적화해야 같은 내용이라도 노출이 완전히 달라진다.

### 1. LinkedIn
- **톤**: 프로페셔널, B2B. "I built..." 1인칭 파운더 관점
- **길이**: 150~300 단어 (800~1,600자)
- **포맷**: 첫 2줄이 hook (210자 이내 — "see more" 전에 보이는 영역). 줄바꿈 많이.
- **언어**: 영어 (한글 번역 별도)
- **특징**: 숫자 강조, 경쟁 비교가 잘 먹힘

**노출 최적화:**
- **해시태그**: 3~5개, 포스트 맨 끝에 배치. 대형(500K+ 팔로워) 2개 + 니치(10K~100K) 2~3개 조합
- **POTAL용 해시태그 풀**: #CrossBorderCommerce #TotalLandedCost #Ecommerce #HSTariffCode #CustomsDuty #InternationalTrade #SaaS #APIFirst #BuildInPublic #StartupFounder #FreeTools #TradeCompliance #DutyCalculator
- **외부 링크**: 본문이 아닌 **첫 번째 댓글**에 넣기 (본문 링크 → 도달 40% 감소)
- **첫 30분 전략**: 게시 후 30분 내 자기 포스트에 15자+ 댓글 달기 → 도달 64% 증가
- **포맷 우선순위**: 캐러셀(PDF) > 이미지 > 비디오 > 텍스트만
- **@멘션**: 관련 인물 3~5명 태그 (10명 넘으면 알고리즘 패널티)
- **게시 시간**: 화~수요일, 오후 1~3시 GMT (글로벌 B2B 최적)
- **뉴스레터**: 구독자에게 인앱+이메일 알림 → 알고리즘 우회 가능

### 2. DEV.to
- **톤**: 기술적, 개발자 친화적. 코드 예시 포함
- **길이**: 500~1500 단어 (7분 읽기가 최적 — 기술 글은 길어야 가치가 있다)
- **포맷**: Markdown. 코드 블록, 헤더 활용
- **언어**: 영어 (한글 번역 별도)
- **특징**: API 사용법, 아키텍처, 기술 결정 과정

**노출 최적화:**
- **태그**: 정확히 4개 (DEV.to 최대), 각 20자 이내
- **POTAL용 태그 풀**: `api`, `webdev`, `javascript`, `python`, `saas`, `tutorial`, `opensource`, `productivity`, `beginners`, `devops`, `ecommerce`, `node`
- **Front matter 필수 작성**:
```yaml
---
title: "제목"
description: "검색에 노출될 설명 (1~2문장)"
tags: api, webdev, saas, tutorial
cover_image: https://potal.app/og-image.png
series: "시리즈명 (관련 글 연결 시)"
canonical_url: https://potal.app/blog/원본URL (크로스포스팅 시)
---
```
- **커버 이미지**: 반드시 포함 (1000x420px). 피드에서 눈에 띄는 효과
- **크로스포스팅**: canonical_url 설정하면 SEO 패널티 없음

### 3. Medium
- **톤**: 에세이/칼럼 느낌. 깊이 있는 사고
- **길이**: 500~1500 단어 (7분 읽기 최적)
- **포맷**: 깔끔한 프로즈. 소제목 활용
- **언어**: 영어 (한글 번역 별도)
- **특징**: 업계 인사이트 + POTAL 자연스럽게 연결

**노출 최적화:**
- **태그**: 5개 (최대치 전부 사용). 대형 2~3개 + 니치 2~3개 조합
- **POTAL용 태그**: `Cross Border Commerce`, `Ecommerce`, `SaaS`, `API Development`, `International Trade`, `Startup`, `Developer Tools`, `Customs`, `Free Tools`, `Build In Public`
- **퍼블리케이션**: 초기에는 기존 퍼블리케이션에 기고 → 큐레이션 확률 증가. 이후 자체 퍼블리케이션 전환
- **페이월**: 무료로 게시 (페이월 → 구글 유기 트래픽 44% 감소)
- **SEO**: 제목과 첫 문장이 자동으로 meta description이 됨. 키워드 자연스럽게 포함
- **게시 시간**: 화~수 오전 10시~오후 1시

## 브랜드 채널 참고 가이드 (자동 생성 아님 — 은태님 직접 작성 시 참고)

아래 4개 플랫폼은 매일 자동 생성하지 않는다. 은태님이 특별한 순간(새 기능 출시, 마일스톤, 유저 피드백, 규제 뉴스 등)에 직접 작성할 때 참고하는 가이드.

### X (Twitter)
- **톤**: 짧고 강렬. 한 문장이 한 트윗
- **길이**: 1~3 트윗 (스레드 가능)
- **해시태그**: 1~2개만 (3개 이상 → 40% 패널티)
- **링크**: 첫 번째 리플라이에
- **핵심**: 비교 숫자가 가장 효과적. "Avalara: $1,500/mo. POTAL: $0."

### Threads
- **톤**: X와 비슷하지만 좀 더 캐주얼
- **길이**: 1~3 포스트
- **토픽 태그**: 1개만 (pill-style)
- **핵심**: Instagram에서 공유하면 도달 4배

### Instagram
- **톤**: 비주얼 중심. 캡션은 짧게
- **길이**: 캡션 50~150 단어. 첫 125자가 "더 보기" 전에 보임
- **해시태그**: 3~5개 또는 캡션 내 키워드 (키워드가 도달 30% 높음)
- **핵심**: Reels(도달 2.25x) > 캐러셀(참여 2.14x) > 단일 이미지

### 디스콰이어트
- **톤**: 한국 스타트업 커뮤니티. 빌더의 솔직한 이야기
- **길이**: 200~500 단어
- **언어**: 한국어
- **핵심**: 홍보성 글보다 진짜 빌딩 과정, 인사이트 공유가 효과적

## 실행 워크플로우

### Step 1: 토픽 결정
1. `content/social-media/daily-posts/` 확인 (최근 글 중복 방지)
2. 없으면 폴더 생성
3. 오늘 날짜 + 카테고리 번호로 토픽 선정
4. 핵심 메시지 한 문장으로 정리

### Step 2: 핵심 콘텐츠 작성
1. 핵심 메시지 기반으로 **LinkedIn 버전**을 먼저 작성 (기준 콘텐츠)
2. 이게 오늘의 "원본"이 됨

### Step 3: 3개 플랫폼 변환
LinkedIn 원본을 DEV.to, Medium에 맞게 변환. 복붙이 아니라 재작성.
- 길이 조절
- 톤 조절
- 포맷 변환
- **플랫폼 고유 태그/해시태그/메타데이터** 각각 다르게 적용
- 위의 플랫폼별 노출 최적화 규칙 반드시 따르기

### Step 4: 한글 번역
영문 콘텐츠에는 전부 한글 번역 포함. 번역은 자연스러운 한국어로.
직역하지 말고 한국어 화자가 읽었을 때 자연스러운 문장으로.

### Step 5: 파일 저장
모든 결과물을 하나의 Markdown 파일로 저장:
```
content/social-media/daily-posts/YYYY-MM-DD_catN_topic-slug.md
```

## 품질 체크리스트

콘텐츠 완성 후 반드시 확인:

- [ ] **금지 표현 검사** — 과장 단어 목록 전부 확인. 하나라도 있으면 다시 쓰기
- [ ] **구걸 표현 검사** — 금지 목록 전부 확인. 하나라도 있으면 삭제
- [ ] **이모지 검사** — 포스트당 3개 이하인가?
- [ ] **숫자 정확성** — session-context.md에 있는 수치만 사용했는가?
- [ ] **플랫폼 적합성** — 각 플랫폼의 톤/길이/포맷을 지켰는가?
- [ ] **태그/해시태그** — 각 플랫폼별 올바른 태그 형식과 개수를 지켰는가?
- [ ] **링크 배치** — LinkedIn은 첫 댓글에 링크
- [ ] **한글 번역** — 영문 콘텐츠 전부 한글 병기했는가?
- [ ] **토픽 중복** — 최근 7일과 겹치지 않는가?
- [ ] **POTAL ONLY** — 외부 주제를 다루지 않았는가?
- [ ] **CTA** — 구걸이 아닌 자연스러운 CTA인가? (potal.app 링크 정도)
- [ ] **API URL 확인** — potal.app/api/v1/ 만 사용 (api.potal.io 등 잘못된 URL 금지)
