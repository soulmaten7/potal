## 2. PDF 템플릿 엔진 생성
app/lib/cost-engine/documents/pdf-generator.ts 파일을 새로 만들어줘.

역할: generateDocuments()의 JSON 결과를 받아서 PDF 바이너리(Uint8Array)로 변환.

구현 요구사항:
- pdf-lib 사용 (import { PDFDocument, rgb, StandardFonts } from 'pdf-lib')
- 5개 문서 타입 전부 지원: commercial_invoice, packing_list, certificate_of_origin, customs_declaration, required_documents
- 공통 함수: createPdfPage(doc, title) — 헤더(POTAL 로고 텍스트 + 문서 제목 + 생성일시), 푸터(페이지 번호 + "Generated by POTAL — potal.app")
- Commercial Invoice 템플릿:
  - 헤더: "COMMERCIAL INVOICE" + invoiceNumber + date
  - 2컬럼: Exporter (왼쪽) / Importer (오른쪽) — 이름, 주소, 국가
  - 테이블: Item | HS Code | Qty | Unit Price | Total — 각 아이템 행
  - 하단: Subtotal, Shipping, Insurance, Grand Total, Incoterms, Currency
- Packing List 템플릿:
  - 헤더: "PACKING LIST" + invoiceNumber
  - 테이블: Package # | Description | HS Code | Qty | Dimensions (LxWxH cm) | Weight
  - 하단: Total Packages, Total Gross Weight, Total Net Weight
- Certificate of Origin 템플릿:
  - 헤더: "CERTIFICATE OF ORIGIN" + certificateNumber
  - Exporter/Importer 정보
  - Origin Country + Criteria (WO/PE/RVC 등) + 설명
  - Origin Declaration 텍스트 블록
- Customs Declaration 템플릿:
  - 헤더: "CUSTOMS DECLARATION" + declarationNumber + type (import/export/transit)
  - Declarant 정보
  - 아이템 테이블: Description | HS Code | Value | Weight
  - 하단: Total Duty, Total VAT, Total Fees, Grand Total
- Required Documents 템플릿:
  - 체크리스트 형식: ☐ Document Name — Description (각 문서)
  - 카테고리별 그룹: Universal / Country-Specific / HS Code-Specific / Conditional

export 함수:
- generatePdf(result: GenerateDocumentResult, options?: { documentType?: string }): Promise<Uint8Array>
- generateMultiPagePdf(result: GenerateDocumentResult): Promise<Uint8Array> — 모든 문서를 하나의 PDF로 합침

## 3. PDF API 엔드포인트 생성
app/api/v1/documents/pdf/route.ts 파일을 새로 만들어줘.

- POST 메서드
- 기존 /api/v1/documents와 동일한 input 스키마 수용
- 내부적으로 generateDocuments() 호출 → generatePdf() 호출
- 응답: Content-Type: application/pdf, Content-Disposition: attachment; filename="POTAL_{documentType}_{date}.pdf"
- 쿼리 파라미터 ?format=base64 지원 (JSON으로 base64 인코딩된 PDF 반환 — 이메일 첨부용)
- 에러 핸들링: 400 (잘못된 입력), 401 (인증 실패), 500 (생성 실패)
- rate limiting: 기존 plan-checker 미들웨어 적용
